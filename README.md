# Coffee Sales MVP

Рабочая MVP-версия серверного веб-приложения для учёта продаж кофе.

## Что реализовано
- Сервер FastAPI
- SQLite база данных
- Работа через браузер
- Продажи
- Справочники напитков и чашек
- Цена у напитка
- Запрет изменения цены напитка, если он уже использовался в продажах
- Запрет удаления использованных напитков/чашек
- Корзина продаж
- Суточный отчёт
- Экспорт отчёта в Excel и PDF
- Экспорт/импорт резервной копии с объединением данных

## Запуск

1. Установить Python 3.11+
2. Открыть папку проекта в терминале
3. Выполнить:

```bash
pip install -r requirements.txt
uvicorn backend.app:app --reload
```

4. Открыть в браузере:

```text
http://127.0.0.1:8000
```

## Файлы
- backend/app.py — сервер, API, база данных
- frontend/index.html — интерфейс
- frontend/style.css — стили
- frontend/app.js — логика интерфейса
- coffee_sales.db — появится автоматически после запуска

## Публикация на Render

### Через GitHub

1. Создайте аккаунт на GitHub.
2. Создайте новый репозиторий.
3. Загрузите туда все файлы проекта.
4. Зайдите на Render.
5. Нажмите `New` → `Web Service`.
6. Подключите GitHub-репозиторий.
7. Render может взять настройки из файла `render.yaml`.

Если настройки вводятся вручную:

Build Command:

```bash
pip install -r requirements.txt
```

Start Command:

```bash
python -m uvicorn backend.app:app --host 0.0.0.0 --port $PORT
```

После публикации Render выдаст ссылку вида:

```text
https://coffee-sales.onrender.com
```

### Важно про базу данных

Сейчас используется SQLite. Для тестового запуска это подходит.
Для настоящей постоянной рабочей версии лучше перейти на PostgreSQL, потому что на бесплатных хостингах локальный файл базы может не сохраняться надёжно после перезапуска сервера.
